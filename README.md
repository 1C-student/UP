# Партийная работа

Информационная система для автоматизации партийной работы, включающей учет членов организации, календаря мероприятий и контроля участия.

## Назначение и область применения

Автоматизация организационных процессов с использованием баз данных является ключевым направлением повышения эффективности управления. Информационные системы позволяют централизованно хранить данные, обеспечивать оперативный доступ к актуальной информации и минимизировать влияние человеческого фактора.

В деятельности политических организаций важное значение имеет систематизация данных о составе участников, планируемых мероприятиях и фактической посещаемости. Отсутствие структурированного подхода к учету приводит к дублированию информации и ошибкам в отчетности.

## Возможности системы

- **Авторизация и регистрация пользователей** - вход в систему с различными уровнями доступа
- **Управление членами партии** - добавление, редактирование и удаление данных о членах организации с возможностью добавления фотографий
- **Создание мероприятий** - планирование и редактирование мероприятий с указанием даты проведения
- **Журнал учета выхода** - отметка участия членов партии в мероприятиях
- **Фильтрация и поиск данных** - поиск по различным критериям (ID, ФИО, Город)
- **Встроенная справочная система** - подробная документация по работе с программой с возможностью экспорта в PDF

## Технологический стек

- **Язык программирования**: C++
- **Фреймворк**: Qt 6
- **СУБД**: PostgreSQL
- **Интерфейс доступа к БД**: Qt SQL (QPSQL)

## Структура базы данных

### Таблица cities (Города)

| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | Идентификатор города (первичный ключ) |
| name | VARCHAR(100) | Наименование города |

### Таблица members (Члены партии)

| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | Идентификатор члена (первичный ключ) |
| full_name | VARCHAR(200) | ФИО члена партии |
| photo_path | VARCHAR(500) | Путь к фотографии |
| city_id | INTEGER | Идентификатор города (внешний ключ) |

### Таблица events (Мероприятия)

| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | Идентификатор мероприятия (первичный ключ) |
| name | VARCHAR(200) | Наименование мероприятия |
| event_date | DATE | Дата проведения |

### Таблица event_log (Журнал учета выходов)

| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | Идентификатор записи (первичный ключ) |
| member_id | INTEGER | Идентификатор участника (внешний ключ) |
| event_id | INTEGER | Идентификатор мероприятия (внешний ключ) |
| visit_date | DATE | Дата отметки |

### Таблица users (Пользователи системы)

| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | Идентификатор пользователя (первичный ключ) |
| login | VARCHAR(50) | Логин пользователя |
| password | VARCHAR(100) | Пароль пользователя |
| role | VARCHAR(20) | Роль пользователя (admin/user) |

## Интерфейс программы

Программа содержит следующие окна:

1. **Форма авторизации** - вход в систему с возможностью регистрации нового пользователя
2. **Форма регистрации** - создание учетной записи нового пользователя
3. **Главное окно администратора** - полный функционал управления данными (вкладки: члены партии, мероприятия, города, журнал выходов)
4. **Главное окно пользователя** - ограниченный функционал (просмотр, отметка участия)
5. **Форма редактирования члена партии** - добавление и изменение данных о членах партии
6. **Форма "О программе"** - информация о версии и разработчике
7. **Форма справки** - подробная инструкция по работе с программой

## Разграничение прав доступа

- **Администратор**: полный доступ к управлению всеми данными (члены партии, мероприятия, города, журнал)
- **Пользователь**: просмотр информации о членах партии, отметка участия в мероприятиях, просмотр своих отметок

## Установка и запуск

### Требования

- Qt 6.x
- PostgreSQL 14+
- Компилятор C++ (MinGW или MSVC)

### Сборка проекта

1. Установите Qt 6 и настройте окружение
2. Создайте базу данных в PostgreSQL и выполните дамп из файла PartyWork.sql
3. Откройте проект в Qt Creator
4. Соберите проект (Build -> Build All)

### Настройка подключения к БД

Параметры подключения задаются в файле database.cpp:

```cpp
db.setHostName("localhost");
db.setPort(5432);
db.setDatabaseName("party_work");
db.setUserName("postgre");
db.setPassword("postgres");
```
